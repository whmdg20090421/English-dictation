# 英语听写 App 完整技术方案 Spec

## Why
打造一款完全支持离线使用的沉浸式英语听写应用。通过在管理员设备上内嵌服务器，配合局域网自动发现和同步机制，解决网络不佳环境下的教学和练习痛点，同时提供媲美顶级应用（如 Duolingo 和 Apple Music）的高质感 UI 体验。

## What Changes
- 初始化 React Native 加 Expo 项目，配置 Expo Router 路由和 NativeWind 样式。
- 引入 `expo-sqlite` 作为本地数据库，构建完全离线的超管、管理员、普通用户三级账号体系（基于 JWT）。
- 使用 `expo-av` 开发核心练习界面：视频播放、半透明字幕遮罩、底部打字作答和结果评分。
- 在管理员设备内嵌 Node.js + Express 服务器，利用 `react-native-zeroconf` 实现局域网广播与自动设备发现。
- 引入 WebSocket 实现管理员与普通设备的实时通信及在线状态监控。
- 构建“离线优先”的数据同步队列：客户端本地保存进度并标记待同步，连接后自动批量上传；管理员端可手动下发题库和视频。
- 整合并转化已有的 Python 评分算法与业务逻辑到应用中。
- 全局 UI 采用深蓝色主调、深色磨砂质感、大圆角与发光边框，结合流畅动画实现沉浸式体验。

## Impact
- Affected specs: 离线账号与权限管理、核心听写练习与评分、局域网设备互联与数据同步、题库及进度管理后台。
- Affected code: 全新项目结构，包含 App 客户端前端、本地 SQLite 数据库、内嵌 Node.js 服务端、WebSocket 通信层。

## ADDED Requirements

### Requirement: 离线账号与三级权限体系
系统 SHALL 提供完全离线的账号管理功能。
#### Scenario: 登录与权限控制
- **WHEN** 用户打开 App 并登录
- **THEN** 系统通过本地 SQLite 验证，签发 JWT。根据角色（超管/管理员/普通用户）渲染不同的界面（普通用户仅练习/记录，管理员附加题库、用户及同步管理）。

### Requirement: 沉浸式听写练习
系统 SHALL 提供视频播放与字幕听写功能。
#### Scenario: 用户答题
- **WHEN** 普通用户进入练习页
- **THEN** 视频开始播放，字幕区域被半透明玻璃态遮罩覆盖。用户在底部输入框输入听到的内容并提交，系统调用评分算法（源自 Python 逻辑）给出正确答案与评分，并计入本地历史记录。

### Requirement: 局域网内嵌服务器与自动发现
管理员设备 SHALL 能作为服务器运行。
#### Scenario: 设备互联
- **WHEN** 管理员打开 App 并进入同步中心
- **THEN** 启动后台 Express 和 WebSocket 服务，并通过 zeroconf 广播。普通客户端自动发现该设备并建立 WebSocket 连接。

### Requirement: 离线优先的数据同步
系统 SHALL 保证断网下的数据完整性并在联网后自动同步。
#### Scenario: 数据上报与推送
- **WHEN** 客户端连接到管理员服务器
- **THEN** 自动将本地队列中“待同步”的练习记录批量发送。管理员确认后客户端更新状态为“已同步”。管理员可主动向指定设备推送新题库。
