# 英语听写 App 完整技术方案 Spec

## Why
在保留原有 Python 版本（移动端听写系统）的所有词库管理、多账户体系、丰富听写模式和详细统计数据的基础上，升级为 React Native 移动端原生应用。不仅提升 UI 的精美度与交互体验（深蓝色沉浸式玻璃态），还附加视频听写模式（遮挡字幕），并利用内嵌服务器实现无网环境下的局域网设备发现与数据同步，打造全场景适用的英语听写利器。

## What Changes
- 框架重构：使用 React Native 加 Expo，配置 Expo Router 路由和 NativeWind。
- 数据迁移：将原 JSON 数据仓转化为 `expo-sqlite` 本地数据库，支持原有的目录/单元/单词（词性释义）结构及统计数据。
- UI 全面升级：严格遵循原版 UI 文字（如“移动端听写系统”、“专属数据详情”、“错题本重练”），采用深蓝主色调、深色磨砂质感和玻璃态卡片（Glassmorphism）设计。
- 完整保留原版功能：多账户切换/管理、多模式听写（拼写、释义、融合试卷）、倒计时与提示系统、详细的错题与历史流水统计、后台词库编辑与导入导出。
- 附加视频听写功能：基于 `expo-av`，支持视频播放、半透明字幕遮罩及听写答题。
- 附加局域网同步功能：在管理员设备内嵌 Node.js + Express 服务，结合 `react-native-zeroconf` 和 WebSocket，实现设备自动发现、离线同步队列和数据批量上报/推送。

## Impact
- Affected specs: UI/UX 重构、听写引擎（词汇+视频双轨）、本地数据架构、局域网网络层。
- Affected code: 全新 RN 项目代码，替代原有的 Python (NiceGUI) 脚本。

## ADDED Requirements
### Requirement: 兼容原版的所有业务功能
系统 SHALL 包含原 Python 脚本的所有功能模块。
#### Scenario: 原版功能复刻
- **WHEN** 用户使用应用
- **THEN** 可以进行账户切换、查看历史正确率与流水、进行多种模式的词汇听写、使用错题本重练，并在管理后台增删改查词库及系统设置。UI 文案与原版保持一致。

### Requirement: 附加视频听写练习
系统 SHALL 提供视频播放与字幕听写功能。
#### Scenario: 视频答题
- **WHEN** 用户进入视频练习页
- **THEN** 视频播放且字幕被半透明玻璃态遮罩覆盖，用户输入听到的内容并提交比对。

### Requirement: 局域网内嵌服务器与自动同步
系统 SHALL 保证断网下的数据完整性并在联网后自动同步。
#### Scenario: 数据上报与推送
- **WHEN** 普通客户端在局域网内发现并连接管理员设备的 Express/WebSocket 服务
- **THEN** 自动将本地队列中的练习记录上报，管理员也可向客户端推送新题库（含词库和视频）。
