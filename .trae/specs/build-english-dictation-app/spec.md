# 英语听写 App 完整技术方案 Spec

## Why
在全面保留原 Python 脚本（基于 NiceGUI 的“移动端听写系统”）的所有细分功能和业务逻辑的基础上，利用 React Native + Expo 重构，打造具备高级沉浸式 UI（深蓝色、玻璃态）、支持断网环境下的局域网设备互联、并拓展了“视频字幕遮罩听写”模式的全新移动端应用。

## What Changes
1. **基础设施**：迁移至 React Native + Expo Router + NativeWind。
2. **数据引擎**：从纯文本 JSON 持久化（`DataManager`）迁移到 `expo-sqlite`。完整复刻数据结构：`vocab` (目录/单元/单词+词性)、`accounts` (含历史流水和单词正确率统计)、`global_settings`。
3. **UI 体系**：100% 还原原版的所有交互文案，采用深蓝色渐变背景 (`linear-gradient(135deg, #0f172a 0%, #2e1022 100%)`)，毛玻璃卡片（`backdrop-filter`），并保留防抖动（shake）及成功/错误闪烁动画。
4. **全量原版功能移植**：
   - **多账户引擎**：新建、重命名、删除、无缝切换，以及设备登录记忆。
   - **主页大盘**：听写次数、练词数、正确率统计，支持“发现保存的进度”（继续进度/开始新听写），以及“错题本重练”（筛选错题并支持勾选）。
   - **数据浏览器 (Data Browser)**：树状展开词库，支持“目录聚合统计”（展示涉及词量、频次及历史会话明细）和“单词精准流水”（展示累计用时及最近50条对错记录）。
   - **听写前置流程**：按层级选择听写内容，选择听写模式（看中拼英、听音拼写、看英写中、融合模式等），设置单题时间和总时间。
   - **核心听写引擎**：支持 TTS 语音播放 (`speakWord`)，基于最长公共前缀 (LCP) 的防碰撞智能提示系统（支持次数限制），单题倒计时与总倒计时双轨监控，允许中途保存进度退出或直接放弃。
   - **评分与结果**：提交后的自动评分、支持对疑似错误的答案进行“人工判卷 (Manual Grade)”，以及最终成绩展示并写入数据库。
   - **管理后台 (Admin)**：
     - *词库管理*：多层级目录增删改名，词条位置上下移动，词条编辑（动态添加多词性及释义）。
     - *导入导出*：支持 JSON 和特定文本格式的词库解析导入与全量导出。
     - *系统设置*：单题时间、允许回退、允许提示、密码修改等。
     - *日志管理*：查看和清空所有个人的听写流水。
5. **附加功能 - 视频听写**：使用 `expo-av` 播放视频，使用毛玻璃层遮挡字幕区域，结合底层听写引擎完成输入与比对。
6. **附加功能 - 局域网同步**：在管理员设备运行 Node.js + Express + WebSocket 服务，使用 `react-native-zeroconf` 广播。普通设备自动连接，通过本地“同步队列”实现离线优先的数据批量上报与题库推送下发。

## Impact
- **Affected specs**: 数据层架构、UI 渲染层、听写业务逻辑层、管理控制台、网络同步层。
- **Affected code**: 全新构建，彻底替换原有的 Python 服务端渲染脚本。
