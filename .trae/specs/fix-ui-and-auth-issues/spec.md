# 修复 UI 和权限验证问题 Spec

## Why
当前英语听写应用存在三个影响用户体验和权限安全的问题：
1. **数据追踪页面标题乱码**：在专属数据详情界面中，标题的字符串插值语法被错误转义（`\${}`），导致用户的名字无法正确渲染。
2. **云端初始化密码配置失效与体验不佳**：配置密码时缺少二次确认和明文查看（小眼睛）功能，容易输错。更严重的是，配置完成后登录管理员会显示“密码错误”，因为在 `cloud_setup_screen.dart` 中调用 `loadData()` 会读取本地的空/旧数据，从而覆盖掉刚在内存中设置的新密码。
3. **添加账户操作不够直观**：目前添加账户的弹窗中使用下拉框选择角色，不符合用户关于“分别点击添加管理员或添加普通用户”的直观预期。

## What Changes
- 修复 `data_browser_screen.dart` 中的标题字符串插值（移除多余的转义符 `\`）。
- 在 `cloud_setup_screen.dart` 中为管理员密码、访客密码、数据加密密钥分别添加“确认密码”输入框。
- 为所有密码输入框添加右侧的小眼睛图标（`suffixIcon`），支持切换密码明文/密文可见状态。
- 修复 `cloud_setup_screen.dart` 中初始化云端时的保存逻辑：先执行 `loadData()` 加载（同步）空/旧数据，再将新加密的密码赋值给 `globalSettings`，最后调用 `saveData()` 将正确密码持久化（同时也兼容“已存在云端”的密码同步逻辑）。
- 重构 `dialogs.dart` 中的 `promptAccountDialog`，将角色选择的下拉框替换为两个并排的明确按钮：“添加普通用户”和“添加管理员”。

## Impact
- Affected specs: 云端初始化逻辑、账户管理逻辑、界面显示。
- Affected code:
  - `lib/screens/data_browser_screen.dart`
  - `lib/screens/cloud_setup_screen.dart`
  - `lib/utils/dialogs.dart`

## ADDED Requirements
### Requirement: 密码输入二次确认与明文查看
系统在配置应用密码和加密密钥时，必须提供双重输入校验，并允许用户通过点击眼睛图标查看密码明文。
#### Scenario: Success case
- **WHEN** 用户在云端初始化页面输入密码并点击小眼睛图标
- **THEN** 密码从隐藏的点变为明文可见状态。
- **WHEN** 用户输入的两次密码不一致并点击提交
- **THEN** 表单验证失败，提示“两次输入的密码不一致”。

### Requirement: 直观的账户角色添加按钮
添加新账户弹窗中提供两个平行的主要操作按钮以区分权限。
#### Scenario: Success case
- **WHEN** 用户点击“新建账户”按钮打开弹窗并输入昵称
- **THEN** 弹窗底部显示“添加管理员”和“添加普通用户”两个按钮，点击其中一个即可直接以对应角色创建账户并关闭弹窗。

## MODIFIED Requirements
### Requirement: 初始化配置保存逻辑
在配置并上传密码配置成功后，应用必须先加载本地和云端的最新同步状态，然后将新密码赋值到全局内存，最后强制保存至本地和云端，确保密码不被旧的空数据覆盖，解决“密码错误”的登录失败问题。
